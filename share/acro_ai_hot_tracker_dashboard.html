系统链路

这不是一个新闻列表，而是一条可维护的信息处理链

MVP 到稳定版的路线
01 数据获取

RSS、Google News、官网公开页、人工补充入口。

02 数据处理

去重、关键词命中、噪音过滤、相关性评分、分层。

03 数据存储

MVP 先用 GitHub JSON；稳定版再考虑数据库和历史检索。

04 数据展现

Dashboard、公司页、日报、主题热度、API / Agent 接入。

待确认事项

给领导确认的不是技术细节，而是边界和优先级

公司范围第一批是 ACRO + 竞品，还是也包含客户、合作伙伴、潜在客户？
推送标准什么内容要即时提醒，什么只进日报，什么只归档？
数据边界LinkedIn、微信、登录平台是人工看，还是后续申请付费/API？
公开性公司名单和结果能否放 GitHub Pages，还是需要内部部署？
展示语言中文、英文、日文，还是给日本团队做双语？
