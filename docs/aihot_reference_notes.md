# AIhot 参考分析

## 观察到的实现形式

AIhot 更像一个 AI 行业动态聚合与精选系统，而不是普通新闻门户。它的核心结构包括：

- 精选信息流
- 全部动态
- AI 日报
- 主题分类
- 收藏
- Agent 接入
- RSS / REST API / Skill 接入

它的产品逻辑是：每天抓取 AI 圈新动态，用 AI 筛掉噪声，把真正值得看的条目留下来，并为每条内容生成摘要、分数、标签和推荐理由。

## 对 ACRO 场景可借鉴的部分

### 1. 精选优先，而不是全量优先

老板和市场部不应该先看到全部新闻，而应该先看到经过筛选的高价值信号。全量候选可以保留，但默认折叠到筛选器里。

### 2. 每条内容都需要推荐理由

只给标题没有价值。ACRO 版本应该展示：

- 为什么相关
- 属于什么主题
- 分数是多少
- 该不该推送
- 对市场部有什么动作建议

### 3. 日报是默认节奏

AIhot 有日报入口。ACRO 版本也应该以日报为基础，只有重大事件才做即时提醒。

### 4. 以后可以开放 Agent/API 接入

AIhot 提供 RSS、REST API、Skill。ACRO 版本早期不必做完整 API，但现在已生成 `data/latest_run.json`，后续可以自然升级成内部 API。

## 不应照搬的部分

AIhot 是面向公共 AI 圈读者的阅读索引；ACRO 版本是企业市场部情报工作台。因此 ACRO 版本不能只做“有趣新闻”，还要多三层：

- 竞品关系：这条新闻和哪些竞品、产品线、市场区域有关
- 业务动作：Newsletter、LinkedIn、展会材料、销售话术、BD 线索
- 管理层视角：哪些信号值得老板看，哪些只是归档

## 当前 dashboard 的取舍

当前原型借鉴了 AIhot 的信息流、精选、主题和日报逻辑，但做了企业化改造：

- 左侧固定导航
- 老板总览指标
- 市场部精选信号
- 主题热度和来源构成
- 行动队列
- 默认只展示日报级内容，避免噪音

下一步如果接入完整目标公司名单，可以扩展为：

- 公司池视图
- 竞品矩阵
- 产品线主题页
- 日本市场专页
- 即时提醒规则配置页
