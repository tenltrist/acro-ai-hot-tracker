# AI Hot Tracker MVP 产品说明

## 一句话

AI Hot Tracker 不是“看到新闻就提醒”的爬虫，而是一个目标公司情报雷达：每天从公开来源收集候选动态，按相关性、重要性、时效性和噪音风险打分，再决定即时提醒、进入日报或只归档。

## 为什么先做 ACRO 标本

ACROBiosystems / 百普赛斯适合作为第一个样本，因为它同时具备几类可验证信号：

- 公司自有新闻和活动页
- PR / 行业媒体转载
- 日本市场相关内容
- ADC、CGT、GMP、organoid、HEK293 等明确业务关键词
- 展会、webinar、产品发布、合作、授权、获奖等可分类事件

如果 ACRO 的规则能跑通，后续可以把休假前公司名单按同一格式接入。

## MVP 不做什么

第一版刻意不做这些：

- 不做全网实时爬虫
- 不接付费新闻 API
- 不自动抓 LinkedIn
- 不承诺分钟级实时监控
- 不因为公司名出现一次就推送

这样是为了把成本和维护风险压低，先验证“什么叫值得推送”。

## 第一版机制

### 0. 更新频率

当前建议频率是每天 1 次，优先放在工作日上午或下班前生成日报。

原因：

- 市场部需要的是可读情报，不是秒级告警。
- 免费 RSS / 官网页面并不保证分钟级更新。
- 每天一次可以显著降低抓取、AI 摘要和人工复核成本。

后续如果确认某些类别必须即时推送，可以只对高价值类别开启更高频率，例如：

- 重大合作 / MOU
- 新产品发布
- 监管 / 临床 / IPO 重要节点
- 竞品进入日本市场

### 1. 候选收集

来源分三类：

- owned：公司官网、新闻页、活动页
- aggregator：Google News RSS
- wire：PR Newswire 等新闻稿来源，可先通过 Google News RSS 间接覆盖

### 2. 规则评分

每条候选会看：

- 是否命中公司别名
- 是否来自公司自有来源
- 是否命中战略主题，例如 ADC、CGT、GMP、HEK293
- 是否命中业务动作，例如 launch、collaboration、license、award
- 是否命中噪音词，例如 price target、forecast、TradingView
- 是否超过默认时效窗口

### 3. 分层推送

- `immediate`：重大信号，适合接即时提醒
- `daily`：进入每日简报
- `archive`：保留但不推送

第一版默认只生成日报。即时提醒先作为标记，不自动发出。

## 为什么不需要一开始烧 API 钱

这个场景最贵的不是“抓取”，而是“让 AI 阅读大量无关网页”。MVP 的省钱思路是：

1. 先用免费 RSS 和官网页面拿候选。
2. 先用规则过滤掉大部分明显噪音。
3. 只把少量高分候选交给 AI 做摘要或业务解释。

换句话说，AI 不负责全网打捞，AI 负责最后一公里判断。

## 当前 ACRO 试跑结果

2026-07-06 试跑结果：

- 抓取候选：43 条
- 抓取错误：0 条
- 即时提醒候选：0 条
- 进入日报：3 条
- 归档不推送：40 条

这个结果说明第一版过滤机制已经能把大量旧新闻、弱相关内容和金融噪音压下去。

## 现阶段能解决的问题

- 给市场部一个每日选题池，而不是人工到处翻官网和 Google。
- 把目标公司相关新闻按相关性和重要性排序。
- 把明显噪音、旧新闻和弱相关内容压到归档。
- 给老板一个总览：今天有没有值得看的信号、主要集中在哪些主题。
- 验证公司名单接入方法：先用 ACRO 跑通，再复制到竞品和目标客户。

## 现阶段不能解决的问题

- 还不是公网 SaaS，不能直接给别人一个互联网域名访问。
- 还没有自动定时任务，需要手动跑脚本，或后续接 cron / GitHub Actions。
- 还没有真正的 AI 深度总结，目前主要是规则评分和公开摘要清理。
- 还没有完整竞品库，需要导入休假前公司名单和关键词。

## 下一步确认问题

真正开始扩大公司名单前，需要确认：

1. 这个系统给谁看：自己、市场部、BD、销售，还是老板？
2. 哪些类别需要即时提醒：合作、融资、产品发布、竞品动作、法规进展？
3. 哪些内容只进日报：展会、webinar、一般市场文章？
4. 每天摘要应该是中文、日文，还是中日双语？
5. 输出位置是 Markdown、Excel、邮件，还是网页 dashboard？

## 扩展到公司名单的方法

每家公司新增两类配置：

- `companies.json`：别名、市场、业务关键词、噪音词
- `sources.json`：官网新闻页、活动页、Google News RSS 查询

当公司数量变多后，再考虑：

- 每家公司单独日报
- 总览 dashboard
- 高优先级公司即时提醒
- 竞品矩阵
- AI 生成业务建议
